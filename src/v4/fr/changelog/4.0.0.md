# 4.0.0

## API

**[API v4.0.0](https://github.com/20syldev/api/releases/tag/4.0.0)** — Réécriture TypeScript complète.

- Migration complète en **TypeScript** avec typage strict.
- Restructuration du projet en modules séparés (`middleware/`, `routes/`, `config/`, `storage/`, `types/`, `utils/`).
- Ajout de l'endpoint **[/health](https://api.sylvain.sh/health)** avec des statistiques techniques (uptime, mémoire, version Node.js).
- Standardisation de toutes les réponses d'erreur avec un helper unifié.
- Ajout du champ `documentation` dans toutes les réponses d'erreur.
- **Node.js >= 22.0.0** requis.

### Réponses enrichies

- **Username** retourne maintenant `number`, `adjective`, `animal` et `job` en plus de `username`.
- **Personal** restructuré avec `vehicle` (array), `social_profiles` (objet), `emergency_contacts` (array d'objets), `subscriptions` et `pets` (arrays).
- **Domain** retourne maintenant `ip_address` et `dns_servers` sous forme de tableaux.

### Mises à jour d'endpoints

- **Convert** — Conversion de température uniquement : Celsius, Fahrenheit et Kelvin.
- **Time** — 17 options de format, type `live`/`random`, 5 fuseaux horaires.
- **Token** — Type par défaut `alphanum`, plage de longueur 12–4096.

## Documentation

- Site de documentation complet construit avec **VitePress**.
- Ajout des composants **Playground**, **Panel** et **Try** pour tester les endpoints directement depuis la documentation.
- Ajout du composant **Examples** avec snippets multi-langages via le composable **useCodeSnippets**.
- Ajout du composant **Banner** et des bannières de migration sur les pages d'index des anciennes versions.
- Ajout du composable **useVersion** pour la gestion des chemins selon la version courante.
- Ajout d'un registre de métadonnées pour les endpoints.
- Réorganisation des styles CSS en structure **base**, **components** et **layout**.
- Réorganisation des utilitaires **i18n**, **icons** et **redirect** dans des sous-dossiers.
- Génération dynamique de la regex des versions à partir de **KNOWN_VERSIONS**.
- Mise à jour du **sélecteur de version** et du **Layout** avec améliorations i18n pour la v4.
- Mise à jour de la **page d'accueil** pour promouvoir la v4 comme version courante.
- Suppression du préfixe de version dans les chemins d'endpoints des v1, v2 et v3.
- Correction d'une **fuite mémoire** dans le composant Terminal via un flag alive.
- Restauration des **accents français** manquants sur la page d'accueil et les anciens index.
- Ajout du guide **VERSIONING.md** pour les futures versions de l'API.
- Ajout des configurations **Prettier** et **ESLint v9** (flat config) avec support Vue et TypeScript.
