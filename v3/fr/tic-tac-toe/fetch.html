API REST : Récupération du morpion

POST /v3/tic-tac-toe/fetch

L'endpoint /tic-tac-toe/fetch permet de récupérer une partie de morpion en cours. Vous pouvez utiliser l'identifiant de la partie en cours pour l'afficher ou créer une nouvelle partie si aucun ID n'est fourni.

Paramètres
  • username (requis) : Le nom d'utilisateur effectuant la requête.
  • game (optionnel) : L'identifiant de la partie à afficher. Si il n'est pas fourni, un nouvel identifiant de partie sera généré.
  • private (optionnel) : Définir à true pour créer une partie privée (défaut : false).
Requête
$ curl -X POST -d "username=Utilisateur1&game=VWXYZ" "https://api.sylvain.pro/v3/tic-tac-toe/fetch" 
{
  "id": "VWXYZ",
  "moves": [
    {
      "username": "Utilisateur1",
      "move": "2-2",
      "session": "abc123"
    },
    {
      "username": "Utilisateur2",
      "move": "1-3",
      "session": "def456"
    }
  ],
  "players": ["Utilisateur1", "Utilisateur2"],
  "turn": "Utilisateur1",
  "status": "ready",
  "private": false
}
Valeurs de statut de partie
  • waiting : La partie a moins de 2 joueurs
  • ready : La partie a 2 joueurs et est prête à être jouée
Réponse de partie terminée
{
  "id": "VWXYZ",
  "moves": [...],
  "players": ["Utilisateur1", "Utilisateur2"],
  "turn": null,
  "status": "finished",
  "private": false,
  "winner": "Utilisateur1",
  "loser": "Utilisateur2",
  "tie": false
}
Si une valeur de username est vide, une erreur sera renvoyée. Lorsqu'une nouvelle partie est créée, l'utilisateur est automatiquement ajouté à la liste des joueurs. Les parties privées ne sont pas visibles dans la liste publique des parties mais peuvent toujours être rejointes si vous connaissez l'ID de la partie. La réponse inclut l'état actuel de la partie, les joueurs, les coups et les informations de tour.
